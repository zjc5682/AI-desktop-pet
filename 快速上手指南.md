# AI 桌面伴侣快速上手

## 当前主工程

本仓库当前实际运行入口是 `apps/pet`。

- 桌面端主程序：`apps/pet`
- Python 本地后端：`backend-python`
- 启动命令优先使用仓库根目录脚本；根目录旧浏览器入口已经删除，不再需要区分旧 `src` / `src-tauri`

## 环境要求

- Node.js 20+
- `pnpm` 9+
- Python 3.11
- Windows 10/11

可选但强烈建议：

- Ollama，用于本地聊天模型
- 本地 `faster-whisper` CTranslate2 模型目录，用于离线 STT

## 首次安装

在仓库根目录执行：

```powershell
pnpm install
```

安装 Python 后端基础依赖：

```powershell
pnpm --filter @table-pet/pet setup:backend
```

按功能补充依赖：

```powershell
pnpm --filter @table-pet/pet setup:backend:memory
pnpm --filter @table-pet/pet setup:backend:voice
pnpm --filter @table-pet/pet setup:backend:perception
pnpm --filter @table-pet/pet setup:backend:screen
```

如果你要一次装全：

```powershell
pnpm --filter @table-pet/pet setup:backend:all
```

## 启动桌面程序

推荐在仓库根目录执行：

```powershell
pnpm dev
```

这会启动 `apps/pet` 的 Tauri 桌面程序。

当前版本已经支持桌面端自动拉起 `backend-python`，正常情况下不需要再手动开一个 Python 终端。

如果你只想单独调试前端或后端：

```powershell
pnpm --filter @table-pet/pet dev
pnpm --filter @table-pet/pet start:backend
```

## 聊天功能配置

设置页里聊天目前支持：

- `Ollama`
- `OpenAI`
- `Zhipu`
- `Qwen Local`

推荐优先级：

1. 已装 Ollama：直接选 `Ollama`
2. 有云端 Key：选 `OpenAI` 或 `Zhipu`
3. 有本地 GGUF：选 `Qwen Local`

如果你选 `Ollama`，需要先确认：

```powershell
ollama list
```

默认模型配置是：

- URL：`http://127.0.0.1:11434`
- Model：`qwen2.5:7b`

## 语音功能配置

当前桌面语音链路：

- 麦克风采集：Rust 原生采集
- STT：`faster-whisper` 本地模型路径
- TTS：`edge-tts` 优先，失败时自动回退到 Windows 系统语音

设置页需要重点确认：

- `Voice Enabled`
- `STT Provider = faster-whisper`
- `STT Model Path = 你的本地 CTranslate2 模型目录`
- `TTS Provider = edge-tts` 或 `system-tts`

建议把本地 STT 模型放到类似目录：

```text
D:\models\faster-whisper\small
```

然后在设置页填这个完整路径。

注意：

- 当前版本不会自动联网下载 `faster-whisper` 模型
- 如果没填本地模型路径，语音识别不会工作

## 摄像头与 Live2D 跟随

要让桌宠自动看着你并跟随人脸：

1. 打开设置页
2. 开启 `Enable camera`
3. 开启 `Face detection`
4. 开启 `Expression recognition`
5. 开启 `Gaze tracking`

现在主宠物窗口在 `cameraEnabled = true` 时会自动启动视觉会话，不再需要额外手动点一次相机按钮。

## 常见问题

### 1. 后端起不来，报 `Directory does not exist`

桌面端现在会从 `apps/pet/src-tauri` 正确回溯到仓库根目录下的 `backend-python`。

如果仍报错，先确认这个目录存在：

```text
<repo>\backend-python
```

### 2. 控制台一直刷 `ws://127.0.0.1:8766 ERR_CONNECTION_REFUSED`

说明本地 websocket 后端没起来。

先看自动拉起是否失败，再手动测试：

```powershell
pnpm --filter @table-pet/pet start:backend
```

### 3. 语音没反应

优先检查：

- Python 3.11 是否可用
- `backend-python` 依赖是否安装完成
- 设置页里 `STT Model Path` 是否填写了本地模型目录
- `TTS Provider` 是否切到 `system-tts` 做排查

### 4. 摄像头开了但角色不跟随

检查：

- 设置里的 `Enable camera` 是否保存成功
- `Gaze tracking` 是否开启
- Python 视觉依赖是否已执行 `setup:backend:perception`
- 摄像头权限是否允许

## 日志位置

桌面端自动拉起的 Python 后端日志会写到应用数据目录下的日志文件。

如果桌面端启动失败，优先看这个日志文件内容。

## 当前建议的最短流程

```powershell
pnpm install
pnpm --filter @table-pet/pet setup:backend
pnpm --filter @table-pet/pet setup:backend:voice
pnpm --filter @table-pet/pet setup:backend:perception
pnpm dev
```

然后在设置页完成：

- 选择聊天提供方
- 填 STT 本地模型路径
- 开启语音
- 开启摄像头与 gaze tracking
