# 项目概况

## 1. 当前项目是什么

这是一个桌面优先的 AI 桌面伴侣项目，当前真实运行链路是：

1. `apps/pet` 提供 Tauri + Vue 3 桌面界面
2. `apps/pet/src-tauri` 提供 Rust 桌面能力
3. `backend-python` 提供本地 WebSocket 后端
4. `packages/*` 提供共享设置、协议、记忆和运行时抽象

聊天后端现在已经支持按请求切换：

- Ollama
- 本地 Qwen
- OpenAI
- 智谱

其中本地 Qwen 仍然需要：

- 有效的 GGUF 模型文件
- `llama-cpp-python`

## 2. 目录框架

```text
.

├─ backend-python/
```

说明：

- 根目录旧浏览器入口已经移除，不再作为运行目标

## 5. 现在应该怎么调试

### 安装依赖

```powershell
pnpm install
pnpm --filter @table-pet/pet setup:backend
pnpm --filter @table-pet/pet setup:backend:voice
pnpm --filter @table-pet/pet setup:backend:perception
pnpm --filter @table-pet/pet setup:backend:memory
pnpm --filter @table-pet/pet setup:backend:screen
```

### 启动

```powershell
pnpm dev
```

说明：

- 主入口是桌面端，不是浏览器
- 本地后端会按需自动拉起

### 常用校验

```powershell
pnpm --filter @table-pet/pet verify:frontend
pnpm --filter @table-pet/pet verify:desktop
pnpm --filter @table-pet/pet verify:backend
pnpm --filter @table-pet/pet verify:all
```

### 单独起后端

```powershell
pnpm --filter @table-pet/pet start:backend
```

### 语音问题排查

先看这几个点：

1. 设置页里的 `STT Provider`
2. `STT Model Path` 是否为本地有效目录
3. `backend-python` 虚拟环境是否已安装语音依赖
4. 桌面端自动拉起后端是否成功

如果语音启动失败，优先查看：

- Tauri 返回的 `ensure_backend_service` 报错
- 应用数据目录里的 `backend-websocket.log`
- 设置页里的 Voice runtime diagnostics

### 聊天问题排查

确认：

1. `defaultChatProvider` 当前选择的是哪个 provider
2. Ollama / OpenAI / 智谱 / 本地 Qwen 的配置是否填对
3. 本地 Qwen 的 GGUF 路径是否有效
4. WebSocket 后端是否正常启动

## 6. 当前开发约束

- 只维护桌面端主链路
- 不要再往根目录旧浏览器入口加功能
- `public/` 仍是当前桌面端共享资源目录，不能误删
