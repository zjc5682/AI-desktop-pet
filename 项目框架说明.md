# 项目框架说明

## 当前真实入口

这个仓库现在是一个桌面优先的 monorepo，真正还在持续开发和运行的主入口是：

- `apps/pet`：Tauri + Vue 3 桌面端
- `backend-python`：本地 Python WebSocket 后端
- `packages/*`：共享能力层和类型层

根目录旧的浏览器入口和旧源码树已经移除。当前仓库只保留桌面端主链路，不再维护那一套历史页面入口。

## 顶层目录

```text
.
├─ apps/
│  └─ pet/                    当前桌面应用
├─ backend-python/            本地 AI / 语音 / 视觉 / 记忆后端
├─ packages/                  共享包
├─ crates/                    预留 Rust crate
├─ public/                    当前桌面端共享资源目录
├─ 快速上手指南.md            当前启动说明
├─ 项目框架说明.md            当前架构说明
└─ 如何添加新功能.md          当前扩展说明
```

## 当前主链路

### 1. 桌面前端

`apps/pet` 是现在实际运行的桌面应用。

- `apps/pet/src/components`
  负责桌宠主界面、聊天页、视频页、设置页等 Vue 组件
- `apps/pet/src/stores`
  负责 Pinia 状态，包括聊天、语音、视觉、截图翻译、桌面服务等
- `apps/pet/src/companion`
  负责把 UI、共享包、后端运行时接起来
- `apps/pet/src/i18n`
  负责设置页和运行态文案
- `apps/pet/src/utils`
  负责桌面命令封装、事件监听、工具函数

### 2. 桌面 Rust 层

`apps/pet/src-tauri` 负责所有真正的桌面能力。

- `apps/pet/src-tauri/src/commands.rs`
  暴露给前端调用的 Tauri 命令
- `apps/pet/src-tauri/src/desktop_services.rs`
  本地后端拉起、桌面服务协调、路径解析
- `apps/pet/src-tauri/src/shortcut_manager.rs`
  快捷键和桌面级事件
- `apps/pet/src-tauri/src/audio_capture.rs`
  底层麦克风采集
- `apps/pet/src-tauri/src/audio_playback.rs`
  底层音频播放

### 3. Python 本地后端

`backend-python` 负责模型推理和运行态服务。

- `backend-python/websocket_server.py`
  桌面端统一 WebSocket 入口
- `backend-python/services/chat_service.py`
  聊天模型路由、联网搜索、提示词拼接
- `backend-python/services/voice_service.py`
  VAD、STT、TTS、语音情绪、语音会话逻辑
- `backend-python/services/vision_service.py`
  摄像头感知、人脸、表情、视线、手势
- `backend-python/services/memory_service.py`
  记忆存储与召回
- `backend-python/services/screenshot_service.py`
  截图、OCR、翻译链路

### 4. 共享包

`packages/*` 放的是跨前后端都要复用的结构。

- `packages/shared`
  设置类型、默认值、共享常量
- `packages/memory`
  记忆层抽象
- `packages/perception`
  感知协议和前端接线
- `packages/providers`
  模型提供方适配抽象
- `packages/realtime`
  实时事件与会话能力
- `packages/live2d`、`packages/stage`、`packages/character`
  桌宠舞台、角色、Live2D 相关能力

## 运行流程

### 应用启动

1. 运行 `pnpm dev`
2. 根脚本进入 `apps/pet`
3. Tauri 启动桌面窗口
4. Rust 层按需确保 `backend-python` 启动
5. 前端通过 `ws://127.0.0.1:8766/` 与 Python 后端通信

### 一次典型请求

以语音会话为例：

1. 前端按钮或唤醒词启动语音会话
2. Rust 或前端采集音频
3. 音频经 WebSocket 发给 `voice_service.py`
4. 后端完成 VAD、STT、聊天、TTS
5. 结果回到前端，驱动字幕、气泡、Live2D 动作和音频播放

## 当前开发时最常用的目录

如果你只想快速定位问题，优先看这些位置：

- UI 和设置页：`apps/pet/src/SettingsApp.vue`
- 桌宠主界面：`apps/pet/src/components/Live2DPet.vue`
- 桌面命令封装：`apps/pet/src/utils/desktopCommands.ts`
- 运行时总线：`apps/pet/src/companion/runtime.ts`
- 桌面 Rust 服务：`apps/pet/src-tauri/src/desktop_services.rs`
- 后端总入口：`backend-python/websocket_server.py`

## 启动与校验

安装依赖：

```powershell
pnpm install
pnpm --filter @table-pet/pet setup:backend
pnpm --filter @table-pet/pet setup:backend:voice
pnpm --filter @table-pet/pet setup:backend:perception
```

启动桌面应用：

```powershell
pnpm dev
```

校验命令：

```powershell
pnpm --filter @table-pet/pet verify:frontend
pnpm --filter @table-pet/pet verify:desktop
pnpm --filter @table-pet/pet verify:backend
pnpm --filter @table-pet/pet verify:all
```

## 维护原则

- 新功能优先接 `apps/pet + apps/pet/src-tauri + backend-python`
- 共享设置和类型优先放 `packages/shared`
- 所有桌面能力先看 Rust 层是否已有实现，不要绕开 Tauri 直接做脆弱方案
- 文档、设置项、默认值、运行时链路要一起更新，不要只改其中一层
