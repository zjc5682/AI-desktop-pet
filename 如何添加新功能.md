# 如何添加新功能

## 先判断功能属于哪一层

新增功能前，先判断它落在哪一层，不要一开始就把逻辑塞进某个 Vue 组件。

- 只影响界面展示：放 `apps/pet/src/components` 或 `apps/pet/src/composables`
- 需要持久化设置：先改 `packages/shared/src/index.ts`
- 需要桌面能力：改 `apps/pet/src-tauri/src`
- 需要模型、OCR、语音、视觉、记忆：改 `backend-python/services`
- 需要跨层共享协议：改 `packages/*`

默认原则是桌面优先。根目录旧版浏览器入口已经删除，新功能只落当前桌面主链路。

## 标准开发顺序

### 1. 先补共享设置和类型

如果功能需要配置项，先改：

- `packages/shared/src/index.ts`

通常要同时做三件事：

1. 在设置类型里补字段
2. 在默认设置里补默认值
3. 如果有枚举或常量，也在这里统一定义

## 2. 再补设置页或界面

桌面 UI 一般落在：

- `apps/pet/src/SettingsApp.vue`
- `apps/pet/src/components/*.vue`
- `apps/pet/src/stores/*.ts`

常见分工：

- 组件负责显示
- store 负责状态、WebSocket、桌面命令
- `apps/pet/src/companion/runtime.ts` 负责把运行时对象统一封装起来

## 3. 需要桌面能力时，先走 Tauri

任何和系统交互有关的能力，都优先接到 Rust 层，不要把浏览器方案当成最终实现。

常用入口：

- `apps/pet/src-tauri/src/commands.rs`
- `apps/pet/src-tauri/src/desktop_services.rs`
- `apps/pet/src-tauri/src/shortcut_manager.rs`
- `apps/pet/src-tauri/src/audio_capture.rs`
- `apps/pet/src-tauri/src/audio_playback.rs`

常见场景：

- 启动或探测本地 Python 后端
- 全局快捷键
- 原生音频采集和播放
- 截图、文件整理、系统状态读取

如果新增了命令，通常还要同时更新：

- `apps/pet/src-tauri/capabilities/default.json`
- 前端的 `apps/pet/src/utils/desktopCommands.ts`

## 4. 需要后端推理时，接 Python 服务

后端统一入口是：

- `backend-python/websocket_server.py`

新增能力时，一般流程是：

1. 在 `services/*.py` 里实现功能
2. 在 `websocket_server.py` 注册消息类型或请求入口
3. 在前端 store 或 runtime 里接入调用

当前常用服务文件：

- `backend-python/services/chat_service.py`
- `backend-python/services/voice_service.py`
- `backend-python/services/vision_service.py`
- `backend-python/services/memory_service.py`
- `backend-python/services/screenshot_service.py`

## 5. 需要多语言时，别忘了文案

设置页和运行态文案不是一个文件。

- 设置页文案：`apps/pet/src/i18n/settingsLocale.ts`
- 运行态文案：`apps/pet/src/i18n/runtimeLocale.ts`

如果新增设置项或新页面文案，至少要同步更新英文和中文，不要只改一个语言。

## 6. 需要记忆、角色、协议共享时，优先放 packages

遇到下面这些情况，不要把结构硬编码在前端页面里：

- 需要跨模块共享的类型
- 记忆召回接口
- 感知事件结构
- Provider 抽象
- 角色或舞台共享模型

优先考虑这些包：

- `packages/shared`
- `packages/memory`
- `packages/perception`
- `packages/providers`
- `packages/realtime`

## 一个完整功能的最小落地清单

以“新增一个桌面能力”为例，通常至少要改这些层：

1. `packages/shared/src/index.ts`
   增加设置字段和默认值
2. `apps/pet/src/SettingsApp.vue`
   增加设置 UI
3. `apps/pet/src/stores/*.ts`
   增加状态和调用逻辑
4. `apps/pet/src/utils/desktopCommands.ts`
   增加桌面命令封装
5. `apps/pet/src-tauri/src/*.rs`
   增加原生命令或系统能力
6. `backend-python/services/*.py`
   如果需要模型或后端推理，则补服务实现
7. `apps/pet/src/i18n/*.ts`
   补文案

## 推荐验证流程

前端类型检查：

```powershell
pnpm --filter @table-pet/pet verify:frontend
```

桌面 Rust 检查：

```powershell
pnpm --filter @table-pet/pet verify:desktop
```

Python 后端检查：

```powershell
pnpm --filter @table-pet/pet verify:backend
```

全部检查：

```powershell
pnpm --filter @table-pet/pet verify:all
```

## 两条强约束

- 不要重新引入根目录旧版浏览器入口或旧源码树
- 不要只改页面不改默认值、文案、运行时接线，否则功能看起来“有开关”，实际上不会生效
